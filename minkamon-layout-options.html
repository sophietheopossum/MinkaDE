Schematic front & centre — two ways

Both options keep the globe. The fork is whose stage the schematic takes: option 1 gives it the centre of the whole page and turns every panel into a wired readout; option 3 keeps the page as-is and rebuilds the thermal panel around a smaller centred schematic. Mockups use the real Eternal Darkness tokens and the real 32/38/30 column split; line routing is indicative, final positions get tuned in QML.

Option 1 — machine centre-stage

schematic swaps places with the globe · every panel ties into it · thermal trends return as a readout strip

CPU → package · GPU → MX450 die (Xe lives in the package anyway) · NETWORK → Wi-Fi module · MEMORY → the soldered LPDDR4X flanking the package. Bottom strip: per-sensor temperature history, back from the dead, wired straight up into the board.

Structure

  • Schematic takes the centre column; heatmap + tags unchanged
  • Globe moves to the old thermal slot, right-bottom
  • Thermal trend charts return as a readout strip under the board
  • CPU / GPU / NET / MEM panels stay where they are

Line plumbing

  • New page-level overlay item draws all leader lines in one Canvas
  • Schematic already computes the mm→px transform — it just exports component anchor points
  • Panel tie points are panel-edge midpoints via mapToItem
  • Lines run through column gutters + a lane above the chassis, so they never fight the board art

Space @ 1920×1080

  • Schematic ≈ 660×450 (today's thermal slot: 544×373)
  • Readout strip ≈ 650×360 below the board
  • Globe ≈ 540 px diameter (was ≈ 655 — keeps 82%)

Trade-offs

  • Leader lines get room to travel — the full eDEX look
  • "Readout" concept covers every subsystem, not just temps
  • Page reads as the machine itself
  • Globe demoted from centrepiece to companion
  • Touches MonitorPage + new overlay + anchor exports (~medium rework)

Option 3 — contained thermal revamp

globe keeps the throne · schematic centred inside THERMAL with corner readouts · one file changes

Page untouched outside THERMAL. Schematic shrinks to make room for four corner readouts; lines are short stubs. CPU / GPU / NET / MEM panels stay unwired — the readout concept only covers temperature sensors.

Structure

  • Only THERMAL changes: schematic centred, four trend readouts in the corners
  • Globe keeps centre-stage untouched
  • Heatmap, tags, MAX header all stay

Line plumbing

  • Everything inside one Canvas — lines, charts and board share a coordinate space
  • No page overlay, no anchor exports, no mapToItem

Space @ 1920×1080

  • Schematic ≈ 380×260 — smaller than it is today (544×373), the corners take their cut
  • Readouts ≈ 150×80 each
  • Globe unchanged at ≈ 655 px

Trade-offs

  • One file (TempsPanel.qml), low risk, quick
  • Globe stays the hero
  • Schematic shrinks to fund the readouts
  • Leader lines are 60–120 px stubs — the concept barely registers
  • Other panels stay untied to the board

My take: option 1. "Front and centre" only pays for itself with an actual centre — the lines need distance to travel to read as wiring rather than decoration, and it's the version where the whole page becomes a diagram of the machine. Option 3 spends its budget shrinking the schematic you just approved. The globe survives option 1 at 82% of its diameter and arguably gains: a spinning world next to a glowing motherboard is a very eDEX pairing.
